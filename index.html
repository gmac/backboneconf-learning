The Simple Lessons of Backbone

Backend / Frontend

Backend is a good place to start.

Model / View Separation

  • Model

  • Manages app data.
  • Notifies of changes.

  • View

  • Captures user input.
  • Renders model state.

M/V is Freeform

Views synchronize through shared models.

Collection / Model

Form a direct mapping of RESTful resources.

Collection GET  /muppets/ .... collection.fetch();
POST /muppets/ .... collection.create();
Model GET  /muppets/1 ... model.fetch();
PUT  /muppets/1 ... model.save();
DEL  /muppets/1 ... model.destroy();

API / Modeling

Don't always match ... that's what parse is for.

{
  "muppets": [
    {"id": 1, "name": "Kermit"},
    {"id": 2, "name": "Fozzy"}
  ]
}
 
Backbone.Collection.extend({
  url: "/muppets",
  parse: function(data) {
    return data.muppets;
  }
});
